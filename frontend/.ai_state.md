# Global Design Standards

### Dropdowns and Form Inputs
- **Curving**: Use `border-radius: var(--r-md)` or similar rounded corners for form fields and dropdowns.
- **Background**: Form inputs and dropdown triggers must use `background: white;`.
- **Border**: Use `border: 1px solid var(--border-strong);` to ensure high visibility.
- **Animations**: Implement `fadeUp` animations for dropdown menus (as seen in `pm-assign.component.scss`).
- **Multi-select**: Use the `multi-select-wrap` pattern with checkboxes and `tag-list` for multiple selections.
- **Font**: Ensure checkbox options and dropdown text explicitly use `font-family: var(--font-sans); font-size: 13px;`.

### Checklist Forms
- **Input Fields**: Apply border styling (`border: 1px solid var(--border-strong)`), `background: white`, and `border-radius: var(--r-md)` directly to the `<input>` element to form a pill, not the wrapper `div`.
- **Action Buttons**: Keep action buttons outside the input pill (separated by `gap: 8px`). Use modern ghost/pill buttons (`background: var(--blue-light)`, `color: var(--blue-text)`) explicitly combining text and icons (e.g., `Add +`), with solid color fill on hover.
